Bu platform, C'nin donanıima yakın, manuel bellek yönetimi felsefesini ve Rust'ın modern, güvenli, ownership tabanlı yaklaşımını aynı anda kavramanız için tasarlanmış ciddi bir mühendislik müfredatıdır.

  • Araçlar: Cargo, rust-analyzer, Clippy, Miri (Undefined behavior detection).

1. Bellek Yönetimi: Stack vs Heap ve Ownership

C: Manuel Bellek Yönetimi & Dangling Pointers
// Belleği manuel ayır, kullan, serbest bırak.
// Hata yapma lüksün var, bedeli Segfault/Memory Leak.
void process_data() {
    int *data = malloc(100 * sizeof(int));
    if (!data) handle_error();

    data[0] = 42;
    
    // free(data) unutulursa: Memory Leak
    // free(data) sonrası kullanılırsa: Use-After-Free
    free(data);
}
Rust: Ownership & RAII (Drop Trait)
// Scope bittiğinde derleyici otomatik temizler.
// Derleyici hata yapmanızı compile-time'da engeller.
fn process_data() {
    // Vec heap'te yaşar, scope sonunda otomatik drop edilir.
    let mut data = vec![0; 100]; 
    data[0] = 42;
    
    // free() çağrısına gerek yok.
    // Scope bittiğinde Drop trait tetiklenir.
} // <-- Bellek burada garanti altında serbest kalır.

2. Hata Yönetimi: Return Codes vs Result/Option

C: errno ve Return Codes
#include <errno.h>
#include <stdio.h>

int divide(int a, int b, int *result) {
    if (b == 0) {
        errno = EDOM; // Domain error
        return -1;    // Hata kodu
    }
    *result = a / b;
    return 0; // Başarı
}

// Kullanım:
int res;
if (divide(10, 0, &res) != 0) {
    perror("Division failed");
    // errno string'e çevrilir
}
Rust: Result<T, E> ve ? Operatörü
use std::num::TryFromIntError;

#[derive(Debug)]
enum MathError {
    DivisionByZero,
    Overflow,
}

fn divide(a: i32, b: i32) -> Result {
    if b == 0 {
        return Err(MathError::DivisionByZero);
    }
    Ok(a / b)
}

// Kullanım (? operatörü ile):
fn main() -> Result<(), MathError> {
    let res = divide(10, 0)?; // Hata otomatik yayılır
    println!("{}", res);
    Ok(())
}

3. String İşlemleri: C-Strings vs Rust Strings

C: Null-Terminated Strings
#include <string.h>
#include <stdlib.h>

void process_string() {
    // Stack'te sabit boyutlu
    char buf[100] = "Hello";
    
    // Heap'te dinamik
    char *dyn = malloc(50);
    strcpy(dyn, "World");
    
    // Uzunluk bulmak: O(n) - null'a kadar say
    size_t len = strlen(dyn);
    
    // Birleştirme: Manuel bellek yönetimi
    char *combined = malloc(len + 6);
    strcpy(combined, dyn);
    strcat(combined, "!");
    
    free(dyn);
    free(combined);
}
Rust: String ve &str
fn process_string() {
    // Stack'te string slice (referans)
    let s: &str = "Hello";
    
    // Heap'te büyüyebilir String
    let mut dyn = String::from("World");
    
    // Uzunluk bulmak: O(1) - metadata'da tutulur
    let len = dyn.len();
    
    // Birleştirme: Otomatik bellek yönetimi
    dyn.push_str("!");
    
    // Format makrosu ile kolay birleştirme
    let combined = format!("{}!", dyn);
    
    // Scope sonunda otomatik temizlenir
}

4. Concurrency: pthreads vs Fearless Concurrency

C: POSIX Threads (pthreads)
#include <pthread.h>
#include <stdio.h>

int counter = 0;
pthread_mutex_t lock = PTHREAD_MUTEX_INITIALIZER;

void* increment(void* arg) {
    for(int i = 0; i < 1000; i++) {
        pthread_mutex_lock(&lock);
        counter++; // Kritik bölge
        pthread_mutex_unlock(&lock);
    }
    return NULL;
}

int main() {
    pthread_t t1, t2;
    pthread_create(&t1, NULL, increment, NULL);
    pthread_create(&t2, NULL, increment, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    printf("Counter: %d\n", counter);
    return 0;
}
Rust: Arc, Mutex ve Threads
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    // Arc: Thread-safe referans sayımı
    // Mutex: Karşılıklı dışlama
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..10 {
        let counter = Arc::clone(&counter);
        let handle = thread::spawn(move || {
            let mut num = counter.lock().unwrap();
            *num += 1;
        });
        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("Counter: {}", *counter.lock().unwrap());
}

7. FFI (Foreign Function Interface)

C: Dinamik Kütüphane (.so/.dll)
// mylib.c
#include <stdio.h>

void say_hello() {
    printf("Hello from C!\n");
}

int add(int a, int b) {
    return a + b;
}

// Derleme (Linux):
// gcc -shared -fPIC -o libmylib.so mylib.c

// Kullanım (dlopen):
#include <dlfcn.h>

int main() {
    void *lib = dlopen("./libmylib.so", RTLD_LAZY);
    void (*say_hello)() = dlsym(lib, "say_hello");
    say_hello();
    dlclose(lib);
    return 0;
}
Rust: C Kütüphanesini Çağırma
// build.rs (derleme öncesi çalışır)
fn main() {
    println!("cargo:rustc-link-lib=mylib");
    println!("cargo:rustc-link-search=native=./");
}

// src/main.rs
extern "C" {
    fn say_hello();
    fn add(a: i32, b: i32) -> i32;
}

fn main() {
    unsafe {
        say_hello();
        let result = add(5, 7);
        println!("Result: {}", result);
    }
}

8. Memory Layout ve Unsafe Rust

C: Bellek Düzeni ve Pointer Aritmetiği
#include <stdio.h>
#include <stdint.h>

struct Point {
    int32_t x;
    int32_t y;
};

int main() {
    struct Point p = {10, 20};
    
    // Bellek adresini al
    uint8_t *ptr = (uint8_t*)&p;
    
    // Byte byte oku (endianness test)
    for(int i = 0; i < sizeof(p); i++) {
        printf("Byte %d: %02x\n", i, ptr[i]);
    }
    
    // Pointer aritmetiği
    int arr[] = {1, 2, 3, 4, 5};
    int *p_arr = arr;
    printf("%d\n", *(p_arr + 2)); // 3
    
    return 0;
}
Rust: Unsafe ve Raw Pointers
use std::mem;

#[repr(C)] // C ile aynı bellek düzeni
struct Point {
    x: i32,
    y: i32,
}

fn main() {
    let p = Point { x: 10, y: 20 };
    
    // Raw pointer oluştur
    let ptr: *const Point = &p;
    
    unsafe {
        // Byte byte oku
        let bytes = mem::transmute::<_, [u8; 8]>(p);
        for (i, byte) in bytes.iter().enumerate() {
            println!("Byte {}: {:02x}", i, byte);
        }
        
        // Raw pointer dereference
        let val = (*ptr).x;
        println!("x: {}", val);
    }
    
    // Slice'tan raw pointer
    let arr = [1, 2, 3, 4, 5];
    let ptr = arr.as_ptr();
    unsafe {
        println!("{}", *ptr.add(2)); // 3
    }
}

Proje Kilometre Taşları

Teori pratiğe dönüşmedikçe öğrenilmez. Her ayın sonunda teslim etmeniz gereken mühendislik projeleri.

Ay 1: Temeller

C: Makefile ile Çok Dosyalı Proje

Hedef: Modüler C projesi yapısı kurmak.

Gereksinimler:

  • En az 3 kaynak dosya (.c) ve 2 header (.h)
  • Makefile ile derleme, temizleme hedefleri
  • Hesap makinesi veya basit bir veri işleme uygulaması

Teslim: GitHub repo + README

Rust: Cargo Workspace

Hedef: Rust modül sistemi ve workspace yapısı.

Gereksinimler:

  • Birden fazla crate içeren workspace
  • Ortak kütüphane crate'i
  • En az 2 binary crate

Teslim: cargo build --all çalışır durumda

Ay 2: Bellek Yönetimi

C: Memory Pool Allocator

Hedef: Kendi bellek yöneticinizi yazmak.

Gereksinimler:

  • Önceden ayrılmış büyük blok üzerinde çalış
  • pool_alloc() ve pool_free() fonksiyonları
  • Linked list ile boş blokları takip et
  • Valgrind ile sızıntı olmadığını kanıtla

Teslim: Benchmark + Valgrind raporu

Rust: Safe Wrapper for C Allocator

Hedef: Unsafe C kodunu güvenli Rust API'sine sarmalamak.

Gereksinimler:

  • Ay 2'deki C allocator'ı unsafe bloklarla kullan
  • std::alloc::GlobalAlloc trait'ini implemente et
  • Güvenli Rust API'si sun (panic yok, Result kullan)

Teslim: FFI entegrasyonu + testler

Ay 3: Sistem Programlama

C: Mini Shell

Hedef: Process yönetimi ve sistem çağrıları.

Gereksinimler:

Teslim: Çalışan shell + demo video

Rust: Async Web Crawler

Hedef: Asenkron programlama ve HTTP istemcisi.

Gereksinimler:
