Sistem Programlama Master Planı

Bu platform, C'nin donanıima yakın, manuel bellek yönetimi felsefesini ve Rust'ın modern, güvenli, ownership tabanlı yaklaşımını aynı anda kavramanız için tasarlanmış ciddi bir mühendislik müfredatıdır.

Öğrenme Metodolojisi

C Yaklaşımı (The Hard Way)

  • Kitap: The C Programming Language (K&R), C Programming: A Modern Approach (King).
  • Araçlar: GCC/Clang, Make, GDB (Debugger), Valgrind (Memory Leak detection), AddressSanitizer.
  • Odak: Pointer aritmetiği, manuel heap yönetimi (malloc/free), segfault analizi, veri yapılarını sıfırdan yazma.

Rust Yaklaşımı (The Safe Way)

  • Kitap: The Rust Programming Language (TRPL), Programming Rust (O'Reilly).
  • Araçlar: Cargo, rust-analyzer, Clippy, Miri (Undefined behavior detection).
  • Odak: Ownership, Borrowing, Lifetimes, Trait sistemi, Concurrency (Fearless concurrency), Unsafe Rust.

Derinlemesine Handbook

İki dil arasındaki mimari ve felsefi farkları gösteren ileri seviye karşılaştırmalar.

1. Bellek Yönetimi: Stack vs Heap ve Ownership

C: Manuel Bellek Yönetimi & Dangling Pointers
// Belleği manuel ayır, kullan, serbest bırak.
// Hata yapma lüksün var, bedeli Segfault/Memory Leak.
void process_data() {
    int *data = malloc(100 * sizeof(int));
    if (!data) handle_error();

    data[0] = 42;
    
    // free(data) unutulursa: Memory Leak
    // free(data) sonrası kullanılırsa: Use-After-Free
    free(data);
}
Rust: Ownership & RAII (Drop Trait)
// Scope bittiğinde derleyici otomatik temizler.
// Derleyici hata yapmanızı compile-time'da engeller.
fn process_data() {
    // Vec heap'te yaşar, scope sonunda otomatik drop edilir.
    let mut data = vec![0; 100]; 
    data[0] = 42;
    
    // free() çağrısına gerek yok.
    // Scope bittiğinde Drop trait tetiklenir.
} // <-- Bellek burada garanti altında serbest kalır.

2. Eşzamanlılık (Concurrency) ve Veri Yarışı (Data Races)

C: pthreads & Manuel Senkronizasyon
#include <pthread.h>

int counter = 0;
pthread_mutex_t lock = PTHREAD_MUTEX_INITIALIZER;

void* increment(void* arg) {
    for(int i=0; i<1000; i++) {
        pthread_mutex_lock(&lock); // Kilitle
        counter++;                 // Kritik bölge
        pthread_mutex_unlock(&lock); // Aç
    }
    return NULL;
}
// Mutex unutulursa: Data Race (Tanımsız Davranış)
Rust: Fearless Concurrency & Arc/Mutex
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    // Arc: Atomik Referans Sayımı (Thread-safe paylaşım)
    // Mutex: Karşılıklı Dışlama (İçeriği korur)
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..10 {
        let c = Arc::clone(&counter);
        handles.push(thread::spawn(move || {
            let mut num = c.lock().unwrap();
            *num += 1;
        }));
    }
    // Derleyici, lock almadan veriyi okumanızı engeller.
}

Proje Kilometre Taşları

Teori pratiğe dönüşmedikçe öğrenilmez. Her ayın sonunda teslim etmeniz gereken mühendislik projeleri.

Ay 2: Bellek Pool Allocator (C)

Standart `malloc` kullanmadan, önceden ayrılmış büyük bir bellek bloğu üzerinde kendi özel `alloc` ve `free` fonksiyonlarınızı yazın. Linked list kullanarak boş blokları takip edin.

Ay 2: Safe Wrapper for C Allocator (Rust)

Ay 2'de C'de yazdığınız allocator'ı Rust'ta `unsafe` bloklar kullanarak sarmalayın (wrap). Rust'ın `std::alloc::GlobalAlloc` trait'ini implemente ederek bunu Rust'ın standart allocator'ı yapın.

Ay 4: Mini Shell (C)

`fork()`, `exec()`, `wait()` ve pipe (`|`) operatörlerini kullanarak kendi komut satırı arayüzünüzü yazın. Arka plan işlemlerini (&) destekleyin.

Ay 4: Async Web Crawler (Rust)

`tokio` ve `reqwest` kullanarak asenkron bir web crawler yazın. URL'leri paralel olarak çekin, HTML'i parse edin (scraper crate) ve yeni linkleri kuyruğa ekleyin.

Ay 6: HTTP/1.1 Server (C)

Soket programlama (sys/socket.h) kullanarak, çoklu thread'ler (thread pool) ile çalışan, statik dosyaları serve eden bir HTTP sunucusu yazın.

Ay 6: FFI Entegrasyonu & CLI Tool (Rust)

C'de yazdığınız HTTP server'ın çekirdek mantığını (veya yüksek performanslı bir C kütüphanesini) Rust'a `bindgen` ile import edin. `clap` kullanarak profesyonel bir CLI tool yazın.

Mühendislik Günlüğü (Debug Journal)

Sistem programlamasında karşılaştığınız segfault'ları, lifetime hatalarını ve çözümlerini buraya not alın. Bu, gelecekteki en büyük başvuru kaynağınız olacaktır.

Notlarınız tarayıcınızın yerel depolamasında (Local Storage) şifrelenmeden saklanır.
